Послідовність команд лаб 2
1. docker-compose build — створення образу для FastAPI додатка.
2. docker-compose up -d — запуск незалежних сервісів Web Server та Database Server у фоновому режимі.
3. docker exec fastapi_lab1-web-1 pip list — перевірка встановлених пакетів всередині контейнера.
4. Використання Bind Mounts для синхронізації локальної папки проєкту з папкою /app у контейнері.
5. Використання прапорця --reload для автоматичного перезавантаження сервера Uvicorn при зміні коду в PyCharm.

лаба 3
1. Створення структури папок app/api та app/schemas.
2. Розділення коду на роутери (APIRouter).
3. Створення entrypoint.sh для керування запуском.
4. docker-compose up --build -d (запуск з новою точкою входу).
5. Тестування CRUD операцій через Swagger UI (localhost:8000/docs).
